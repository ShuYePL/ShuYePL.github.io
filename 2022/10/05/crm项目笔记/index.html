    crm 项目简介

    Customer Relationship Manager 客户管理系统：是一种企业级项目，或者说传统项目，维护客户关系的一种项目。

    crm 项目核心业务

    • 系统管理功能：不直接处理业务数据，为了保证业务管理功能的正常安全运行而设计的功能
      • 用户登录
      • 安全退出
      • 登录验证
      • ……
    • 业务管理功能：处理业务数据

    数据库表

    • 主键：数据库表中唯一确定一条记录的一组字段，推荐使用没有业务含义的一个字段做主键。主键字段的类型和长度由主键的生成方式来决定。
      • 主键值的生成方式
        • 自增：借助数据库自身主键生成机制实现，类型是数值型的，长度由数据量来决定，特点是运行效率低，而开发效率高
        • assighed：程序员手动生成，是唯一非空的主键值，使用特定的算法来实现（主要使用的）
          • hi/low ：数值型，长度由数据量决定
          • UUID ：字符串，定长，32位
        • 共享主键：由另一张表的类型和长度决定（不推荐，耦合度太高）
        • 联合主键：由多个字段的类型和长度决定（不推荐）
    • 外键字段：用来确定表和表之间的关系。
      • 一对多：一的那张表叫做父表，多的那张表叫做字表
        • 添加数据时，先添加父表记录，再添加字表记录
        • 删除数据时，先删除字表记录，再删除父表记录
        • 查询数据时，可能会进行关联查询
          • 关联查询的一点小内容：内连接和外连接
            • 内连接：查询所有符合条件的数据，并且要求结果在两张表中都有相对应的记录。
            • 外连接：分为左外连接、右外连接和全外连接。以左外连接为例，查询左侧表中所有符合条件的数据，即使在右侧表中没有相对应的记录。
            • 内连接和外连接的选择
              • 如果外键不能为空，优先使用内连接（内外连接结果都一样，但是内连接效率更高）
              • 如果外键可以为空
                • 假如要查询的内容在两张表中都有相对应的记录，使用内连接
                • 假如只要查询在其中一张表中所有符合条件的就，使用左外连接
      • 一对一
        • 共享主键（不推荐）
          • 添加数据：先添加先产生的表记录，再添加后产生的表记录
          • 删除数据：先删除后产生的表记录，在删除先产生的表记录
          • 查询数据：无需进行连接查询
        • 唯一外键：在外键上加上唯一性约束
      • 多对多：三张表，两主键
        • 添加数据时，先添加父表的记录（两张父表），再添加字表的记录（多出来的一张表）
        • 删除数据时，先删除字表的记录，在删除父表的记录
        • 查询数据时，可能会进行关联查询
    • 关于日期和时间的字段：都按照字符串来处理，处理成定长的字符串
      • 不带时分秒的格式：char(10) “yyyy-MM-dd”
      • 带时分秒的格式：char(19) “yyyy-MM-dd HH:mm:ss”

    配置文件

    各种配置文件某种意义上来讲的话都是可以复制粘贴的，只要看得懂配置文件的内容，当需要修改配置文件的时候知道怎么修改文件就可以了。

    资源存放位置

    页面资源可以存放在 webapp 的 WEB-INF 文件下，起到一定的保护作用，而照片和 js 等文件则可以在直接在 webap 文件下创建文件夹直接存放，不需要放入到 WEB-INF 文件下。

    整个项目写的顺序

    从顶层往底层写，有大局观，被依赖的先写。

    一些注意事项：

    • IDEA 默认的 html 文件的编码方式和 jsp 文件的编码方式是不一样的，所以在把 html 文件改造成 jsp 文件之前需要先改变 html 文件的编码方式，防止中文在改变编码格式之后变成乱码，具体的步骤就是：

      • 将 html 文件最上面的一行 <!DOCTYPE html> 改成 <%@ page contentType="text/html;charset=UTF-8" language="java" %>
      • 然后将文件的 .html 后缀改成 .jsp

    • controller 方法上面填写的路径要和 controller 方法处理完请求后，响应信息返回的页面的资源目录保持一致（也就是说，这个 controller 方法控制的请求是对应与哪一个目录下的 html 文件的，url 就要对应到相应的目录下，然后在目录下的路径名字就是方法的名称了）。

      • 例子：/settings/qx/user/toLogin.do

    • 请求是发同步的还是发异步的判断

      • 如果全局刷新页面，就同步请求
      • 如果局部刷新页面，就异步请求
      • 如果不确定，就异步请求

    • 如果需要 HttpServletRequest 对象的话，可以直接在方法的参数中添加上，ssm 会自动注入这个对象，像这样的对象还有 HttpServletResponse, HttpSession, Model, ModelMap。

    • 公共使用的类（例如携带返回数据的类），可以将其放在对应项目包下的 commons/domain 中，commons 包放公共的资源。

    • 返回的数据如果有很多地方可能会用到的话，就可以定义为 String 类型，如果不清楚要返回数据的类型，就可以将其定义为 Object 类。

    • submit 按钮发送的请求是同步请求，如果是要求局部刷新的地方（像是登录界面），需要将按钮的 type 属性值改成 button ，实现异步请求。

    • 页面向服务器发送数据时，可以向 html 标签添加 id ，然后获取其 jQuery 对象，再利用 jQuery 来完成数据上传等工作，在处理用户上传的账户和密码，可以在利用 jQuery 将账号和密码的数据前后空格去掉，这样可以避免一些错误的发生。

    • 使用 jQuery 获取指定元素的指定属性的值

      • 选择器.attr(“属性名”); 用来获取那些值不是 true/false 的属性的值
      • 选择器.prop(“属性名”); 用来获取值是 true/false 的属性的值
        • 这些属性目前学到的有：checked, selected, readonly, disabled

    • ajax 请求中数据的名字要跟接收参数的 controller 中相应方法的参数名称一样，这样参数才能被接收到。
